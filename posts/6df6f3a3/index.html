专为软件著作权申请设计的源代码文档自动生成工具。项目目标是将源代码文件批量或单个地转换为符合申请要求的 Word (.docx) 文档，自动处理分页、格式与页眉，简化人工排版工作。更多演示与使用方法可查看 B 站教程：https://www.bilibili.com/video/BV1DMGpzbEqr/?spm_id_from=333.1387.homepage.video_card.click&vd_source=33afd104a41faf98cb6a859069c293b4

项目仓库： https://github.com/LeastBit/SourceCodeDoc

项目简介

本工具可解析多种文本型源代码文件（如 JavaScript、Python、Java、C/C++、C#、PHP 等），保留代码结构与原始格式，按配置生成标准 Word 文档，适配软件著作权材料提交的常见格式要求，支持 GUI 与 CLI 两种使用方式，兼顾可视化便捷与脚本自动化。

功能特点

核心功能包括智能分页（按行数或“前后30页”模式处理超长文档）、格式优化（自动调整字体、行距、缩进以满足页面限制）、页眉定制（可插入项目名或版本信息）、批量处理（文件夹递归或单文件输入）与拖拽支持（GUI）。界面选择上提供基于 Electron 的图形界面和可集成到流水线的命令行界面。输出为标准 .docx，保留原始代码顺序并支持页码控制与节选择。

快速开始

环境要求为 Node.js >=14，以及 npm 或 yarn，支持 Windows、macOS 和 Linux。安装与运行示例：克隆仓库 git clone https://github.com/LeastBit/SourceCodeDoc.git 并进入项目目录后运行 npm install；启动 GUI：npm start；查看 CLI 帮助：node cli.js –help。

使用指南

图形界面使用时通过“选择文件/目录”或拖拽添加源文件，配置每页行数、页眉文本与页码模式后点击生成；命令行使用遵循语法 node cli.js <输入路径> [选项]，主要参数为 –output 指定输出文件，–lines 每页行数，–header 页眉文本，–mode 页码模式（all 或 partial），默认每页 50 行，默认页眉“源程序V1.0”，默认模式 all。为处理大型文件建议使用 partial（前后30页）或分批生成。
